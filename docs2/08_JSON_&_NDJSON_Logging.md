# `{ ... }` JSON & NDJSON LoggingLogSmith treats JSON as a first-class output format. Whether you�re building ingestion pipelines, debugging complex systems, or feeding logs into ELK/Loki/BigQuery, LogSmith gives you clean, structured JSON and NDJSON with zero configuration overhead.This chapter explains how JSON and NDJSON logging works, how to enable it, and how to customize the structure.---## 💡 Why JSON Logging MattersText logs are great for humans.  JSON logs are great for machines.JSON / NDJSON gives you:- structured fields  - predictable keys  - easy parsing  - compatibility with log processors  - clean ingestion into databases  - consistent formatting across handlers  LogSmith supports both **pretty JSON** (for humans) and **NDJSON** (for machines).---## 🔹 Output ModesEach handler chooses its own output mode:- **COLOR** — structured text with ANSI  - **PLAIN** — structured text without color  - **JSON** — pretty‑printed JSON  - **NDJSON** — compact JSON, one object per line  Example:```pythonlogger.add_console(output_mode = OutputMode.JSON)logger.add_file(    log_dir      = "logs",    logfile_name = "events.ndjson",    output_mode  = OutputMode.NDJSON,)```---## 🔹 JSON Output (Pretty)Pretty JSON is ideal for debugging or piping into tools like `jq`.Example console output:```json{  "timestamp": "2026-02-15T01:08:46.035Z",  "level": "INFO",  "logger": "demo",  "message": "User login",  "fields": {    "username": "Gilad",    "action": "login"  },  "file": "auth.py",  "line": 42,  "thread": 1234,  "process": 5678}```JSON output includes:- timestamp  - level  - logger name  - message  - structured fields  - optional metadata (file, line, thread, process)  - exception info (if present)  ---## 🔹 NDJSON Output (Compact)NDJSON is the preferred format for ingestion pipelines:- one JSON object per line  - no whitespace  - no indentation  - safe for streaming  - easy to batch process  Example:```{"timestamp":"2026-02-15T01:08:46.035Z","level":"INFO","logger":"demo","message":"User login","fields":{"username":"Gilad","action":"login"}}```NDJSON is perfect for:- ELK  - Loki  - BigQuery  - Datadog  - CloudWatch  - FluentBit  - Kafka  - Any streaming pipeline  ---## 🔹 Structured Fields (Named Arguments)Named arguments automatically become JSON fields:```pythonlogger.info("User login", username = "Gilad", action = "login")```JSON:```json"fields": {  "username": "Gilad",  "action": "login"}```NDJSON:```"fields":{"username":"Gilad","action":"login"}```No extra configuration required.---## 🔹 Exceptions in JSONExceptions are structured too:```pythontry:    1 / 0except Exception:    logger.error("Division failed", exc_info = True)```JSON output includes:```json"exception": {  "type": "ZeroDivisionError",  "message": "division by zero",  "traceback": [    "File \"main.py\", line 10, in <module>",    "1 / 0"  ]}```NDJSON includes the same structure in compact form.---## 🔹 Customizing JSON StructureJSON output uses the same `LogRecordDetails` object as text output.Example:```pythondetails = LogRecordDetails(    datefmt = "%Y-%m-%d %H:%M:%S",    optional_record_fields = OptionalRecordFields(        logger_name = True,        process_id  = True,    ),    message_parts_order=[        "logger_name",        "process_id",        "level",    ],)logger.add_console(    output_mode        = OutputMode.JSON,    log_record_details = details,)```---## 🔹 JSON + RotationJSON and NDJSON work seamlessly with rotation:```pythonlogger.add_file(    log_dir        = "logs",    logfile_name   = "events.ndjson",    output_mode    = OutputMode.NDJSON,    rotation_logic = RotationLogic(maxBytes = 50_000, backupCount = 5),)```Rotated files remain valid NDJSON.---## 🔹 JSON + Async LoggingAsyncSmartLogger supports JSON and NDJSON identically:```pythonlogger = AsyncSmartLogger("async.json", level = 10)logger.add_file(    log_dir      = "logs",    logfile_name = "async.ndjson",    output_mode  = OutputMode.NDJSON,)```Async logging guarantees:- ordered JSON events  - non-blocking writes  - async rotation scheduling  ---## 🔹 JSON + Raw OutputRaw output bypasses formatting and is **not** JSON:```pythonlogger.raw("This is raw text")```Use raw output only for:- banners  - headers  - gradient art  - debugging dumps  Not for structured logs.---## 📘 SummaryJSON and NDJSON logging in LogSmith gives you:- pretty JSON for debugging  - NDJSON for ingestion pipelines  - structured fields  - structured exceptions  - customizable metadata  - rotation support  - async support  - predictable, machine-friendly output
